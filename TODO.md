## Запланировано

### Несколько тейк-профитов (multi-TP)

**Ограничение:** максимум 3 тейка. Выбор: 2 или 3 тейка (переключатель/список).

**Распределение объёма (настраивается в панели настроек):**
- **Равный объём** — на каждый тейк одинаковая доля объёма позиции (лоты); профит по тейкам может отличаться. Пример: 0.03 лот, 3 тейка → по 0.01 лот на каждый TP.
- **Равный профит** — объём по тейкам подбирается так, чтобы примерный профит в деньгах (валюта депозита) на каждом тейке был одинаковым. Расчёт только по цене и объёму (без комиссии/свопа на первом этапе).

**Задание уровней цен TP1, TP2, TP3:**
- **Режим не быстрый ордер (Limit):** есть один тейк (линия/уровень), который пользователь ставит как обычно. Остальные тейки (1 или 2) бот размещает на равном расстоянии друг от друга между лимитным ордером (Entry) и этим TP. Пользователь потом может перетащить линии куда нужно перед нажатием кнопки размещения.
- **Режим не быстрый ордер (Market):** то же — один основной TP, остальные на равных расстояниях между Entry и этим TP; можно перетащить перед размещением.
- **Режим быстрый ордер (Limit или Market):** пользователь по очереди размещает ордера (кликами задаёт каждый TP).

**Где настраивать и что показывать:**
- Настройки (количество тейков 2/3, режим «равный объём» / «равный профит») — в панели настроек (вкладка «...»).
- На основной панели — отображение выбранного режима: блок «Макс. риск %» и поле ввода сократить вдвое (половина ширины), рядом добавить компактное отображение: количество тейков и режим (равный объём / равный профит).

---

**Этапы реализации:**

1. **Панель настроек: выбор количества тейков и режима**
   - Переключатель/список: 2 или 3 тейка.
   - Выбор режима: «Равный объём» / «Равный профит».
   - Сохранение выбора (доступно из основной логики/COP).

2. **Основная панель: отображение текущих настроек TP**
   - Блок «Макс. риск %» + поле ввода — сократить до половины ширины.
   - Рядом: компактный блок с отображением «Тейков: 2» или «Тейков: 3» и «Объём» / «Профит» (режим распределения).

3. **Уровни TP: не быстрый ордер (Limit и Market)**
   - Один основной уровень TP задаётся как сейчас (линия или поле). Остальные уровни (TP2, при 3 тейках — TP2 и TP3) вычисляются: равные расстояния между Entry и этим TP. Рисуются линии/поля; пользователь может перетащить их перед размещением.

4. **Уровни TP: быстрый ордер**
   - Режим Fast Order: пользователь по очереди кликами задаёт каждый тейк (TP1, TP2, при 3 тейках — TP3). Логика аналогична текущему быстрому ордеру, но с 2 или 3 точками TP.

5. **Расчёт объёма на каждый тейк**
   - Режим «равный объём»: объём на тейк = общий объём / N (N = 2 или 3).
   - Режим «равный профит»: по заданным уровням TP и Entry вычислить лоты так, чтобы профит в деньгах на каждом тейке был примерно одинаковым (только цена и объём).

6. **Размещение ордера с несколькими TP**
   - Интеграция с текущим потоком (Limit/Market, риск, линии): одна позиция с частичным закрытием по уровням TP1, TP2, (TP3) или использование API cTrader для нескольких TP. Реализация по возможностям платформы.

7. **Линии на графике и поля панели**
   - Отображение 2 или 3 линий TP и при необходимости полей ввода цен TP; подписи объёма/профита по тейкам при «равный профит».

8. **Документация и проверка**
   - Обновить DOCS.md (и при необходимости README). Проверить сценарии: 1 TP (как сейчас), 2 TP, 3 TP; Limit/Market; Fast Order и обычный режим.
