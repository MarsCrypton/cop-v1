# Главная панель COP v1: структура и отступы

Документ фиксирует разметку **развёрнутой** главной области `MainPanel` (`cop v1/UI/MainPanel.cs`, `PanelStyles.cs`). Отдельных вкладок `TabControl` нет: «главный экран» — это видимый `_contentStack` под шапкой; блок настроек открывается кнопкой Set и не входит в этот список.

## Масштаб и система координат

- Базовая ширина панели: `BasePanelWidth = 228` логических px; на экране `PanelWidth = S(228)` (`PanelStyles.cs`).
- Отступы и толщины из кода задаются через `PanelStyles.ST(...)` и `S(...)` — при **масштабе UI 80–150%** (`SetScalePercent`) значения умножаются на `UiScale = ScalePercent / 100`.
- **Базовые числа ниже — при масштабе 100%.** Для другого масштаба: `значение ≈ базовое × (ScalePercent / 100)`.

## Корневая карточка (вне 12 блоков)

У `_rootWrapper` (`Border`):

- Внешний отступ от края графика: `Margin = ST(8)` со всех сторон.
- Рамка: `BorderThickness = ST(1)`.
- На внутреннюю раскладку шириной `PanelWidth` это не накладывает горизонтальные margin у строк контента — это смещение/обводка всего виджета.

## Иерархия (развёрнутая панель)

`_mainStack` (вертикально):

1. `headerGrid` — шапка.
2. `_collapsedChromeStack` — **скрыт** при развёрнутой панели.
3. `_contentStack` — основной контент.

У `_contentStack` и `_mainStack` нет собственных горизонтальных margin — отступы задают дочерние элементы.

## 12 элементов сверху вниз

| № | Элемент | Описание |
|---|---------|----------|
| 1 | `headerGrid` | Заголовок, кнопка настроек (Set), кнопка Full/Mini. |

| 5 | `riskRow` | Блок риска (подпись, комбо %/USD/EUR, поле, единица, ошибка). |
| 6 | `sep2` | Разделитель. |
| 7 | `_priceLabel` | Подпись лимитного ордера. |
| 8 | `_priceTextBox` | Поле цены (read-only). |
| 9 | `sep3` | Разделитель. |
| 10 | `slTpRow` | Две колонки SL и TP. |
| 11 | Разделитель | Вызов `CreateSeparator()` перед кнопкой. |
| 12 | `_submitButton` | Кнопка размещения ордера. |

## Горизонтальные отступы (Margin Left / Right)

Отсчёт от левого и правого края **области контента шириной `PanelWidth`** (родитель без боковых padding). Значения — **при 100% масштаба UI**.

| № | Элемент | Margin Left | Margin Right |
|---|---------|-------------|--------------|
| 1 | Шапка `headerGrid` | 0 | 0 |
| 2 | `checkboxRow` | 2 | 2 |
| 3 | `modeRow` | 4 | 4 |
| 4 | `sep1` | 0 | 0 |
| 5 | `riskRow` | 2 | 2 |
| 6 | `sep2` | 0 | 0 |
| 7 | `_priceLabel` | 2 | 2 |
| 8 | `_priceTextBox` | 2 | 2 |
| 9 | `sep3` | 0 | 0 |
| 10 | `slTpRow` | 2 | 2 |
| 11 | Разделитель | 0 | 0 |
| 12 | `_submitButton` | 2 | 2 |

### Примечания

- Разделители создаются методом `CreateSeparator()`: `Margin = ST(0, 2, 0, 2)` — горизонтально **0**, вертикально **2** сверху и снизу; `HorizontalAlignment.Stretch` — линия на всю ширину родителя.
- У `_submitButton` в конструкторе сначала задаётся `Margin = ST(4, 4, 4, 6)`, затем `ApplySubmitButtonStyle(..., 0)` **перезаписывает** margin на `ST(2, 6, 2, 2)` — итог по горизонтали **2 / 2**.
- У `_priceTextBox` после инициализатора вызывается `ApplyTextBoxStyle`, который задаёт `Margin = ST(2)` — итог **2 / 2**.

## Смысловые блоки без разделителей

Если считать «модулями» только функциональные зоны (без отдельных линий):

- Шапка  
- Fast Order + спред  
- Режимы Limit / Market  
- Риск  
- Цена входа (подпись + поле)  
- SL / TP  
- Кнопка отправки  

Плюс четыре разделителя между секциями — в таблице из 12 они вынесены отдельными строками.

## Вложенные отступы (не входят в таблицу из 12 строк)

Внутри строк свои margin у дочерних контролов (например, чекбокс Fast Order `ST(4, 4, 8, 4)`, комбо риска `ST(4, 2, 2, 2)`, стили подписей и полей). Для точного позиционирования текста/контрола от края панели их нужно **суммировать** с margin строки-родителя.
