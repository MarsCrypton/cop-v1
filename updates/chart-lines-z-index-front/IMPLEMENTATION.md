# Линии Entry / SL / TP поверх фигур графика (ZIndex)

## Связь

- **Linear:** [MAR-78](https://linear.app/marskorshun/issue/MAR-78/updates-linii-entrysltp-poverh-figur-grafika-zindex)
- **Проект Linear:** Плагин COP (лимитки)
- **Ветка Git (предложение):** `mar-78-updates-linii-entrysltp-poverkh-fihur-hrafika-zindex`

## Зачем это обновление

На графике cTrader пользователь может нарисовать **фигуры** (прямоугольники, зоны и т.п.), которые визуально и по **порядку слоёв** оказываются **над** линиями робота. В результате при совпадении уровня с TP или SL **выделяется фигура**, а **перетащить линию** бота сложно или невозможно.

**Цель:** сделать торговые линии и подписи COP **выше по Z-оси** относительно обычных объектов с низким приоритетом, чтобы линии оставались **активными для перетаскивания** в типичной ситуации.

## Решение в коде

1. **`PanelStyles.cs`** — константы:
   - `ChartTradingLineZIndex` (`1_000_000`) — все горизонтальные линии Entry / SL / TP, которые рисует COP.
   - `ChartTradingLabelZIndex` (`1_000_001`) — текстовые подписи у линий (чуть выше линии, чтобы текст читался поверх линии).

2. **`ChartLineManager.cs`**
   - После `DrawHorizontalLine`: `line.ZIndex = ChartTradingLineZIndex` (вместе с уже существующим `IsInteractive`).
   - После `DrawText`: `IsInteractive = false`, `ZIndex = ChartTradingLabelZIndex` — подпись не должна **перехватывать** интерактив там, где платформа это учитывает.

3. **`FastOrderHandler.cs`** — те же правила для быстрых линий и их подписей (`DrawFastLine` / `DrawFastText`), чтобы поведение совпадало с основным режимом.

## Ограничения (важно для теста)

- У объектов графика в cAlgo есть **`ChartObject.ZIndex`**, но если пользователь **вручную** поднимет фигуру выше (если терминал это позволяет), приоритет может снова оказаться у фигуры.
- **Гарантии «всегда поверх любого объекта»** API не даёт — мы поднимаем слой **на типичный уровень** относительно дефолтных рисунков.
- Узкая **полоса линии** всё равно требует точного попадания; при необходимости можно отдельно обсуждать увеличение `LineThickness` (визуальный компромисс).

## Проверка

- Включить режим с линиями (Limit / Market), наложить на уровень TP/SL **прямоугольник** с графика — попытаться **схватить зелёную/красную линию** и перетащить: ожидается, что чаще срабатывает именно линия COP.
- Fast Order: те же линии не должны уходить «под» фигуры сильнее, чем раньше.
