# COP v1 — тёмный UI с оранжевым акцентом (референс мини-панели)

## Назначение документа

Зафиксировать визуальный язык целевого макета (скрин «COP V1 — MINI PANEL»): цвета, типографика, скругления, отступы и структура блоков. На этой ветке (`feature/ui-dark-orange-mini-panel`) дальше можно безопасно внедрять изменения; откат — через `git checkout main` или merge/revert.

---

## Референс: структура экрана

```
┌─────────────────────────────────────────────┐
│ ●  COP V1 — MINI PANEL          (шапка)     │
├─────────────────────────────────────────────┤
│ [ LM ] [ MK ] [ OK ] [ FST ]   (кнопки)     │
├─────────────────────────────────────────────┤
│ Свернутая панель          ● Активна (футер) │
└─────────────────────────────────────────────┘
```

| Зона | Содержимое |
|------|------------|
| Шапка | Слева: круглый маркер-акцент (оранжевый). Справа от него: заголовок капсом, приглушённый серый. |
| Ряд кнопок | Четыре равной ширины кнопки с небольшим зазором. Активная — заливка оранжевым; неактивные — тёмный фон + тонкая обводка. |
| Разделитель | Тонкая горизонтальная линия между кнопками и футером. |
| Футер | Слева подпись состояния; справа индикатор (зелёный круг) + текст статуса. |

## Цветовая палитра (hex)

Значения взяты с референса; при расхождении на мониторах допустима ±коррекция на 1–2 шага яркости.

| Роль | Hex | Назначение |
|------|-----|------------|
| Фон панели (внешний) | `#0D0D0D` | Основной фон контейнера мини-панели / более глубокий слой. |
| Фон неактивной кнопки / внутренний | `#1A1A1A` | Подложка неактивных кнопок, «карточный» уровень. |
| Акцент (активная кнопка) | `#F25F00` или `#FF6600` | Основной оранжевый; hover — чуть светлее/насыщеннее (например `#FF7722`). |
| Статус «вкл» | `#00C853` | Круг индикатора «Активна». |
| Текст основной | `#FFFFFF` | Подписи на кнопках (жирный). |
| Текст приглушённый | `#666666` | Заголовок шапки, футер, вторичные подписи. |
| Разделитель | `#1F1F1F` | Линия между рядом кнопок и футером. |
| Обводка неактивной кнопки | `#333333`–`#404040` (оценка) | Тонкая граница для отделения от фона `#0D0D0D`; уточнить визуально в cTrader. |

### Сопоставление с текущим кодом (`PanelStyles.cs`)

| Сейчас | Референс (цель) |
|--------|-----------------|
| `PanelBackground` `#2D2D2D` | Углубить до `#0D0D0D` (и ветка прозрачности — от этого базового цвета). |
| `ButtonActive` `#00BFA5` | Заменить на оранжевый акцент (`#F25F00` / `#FF6600`). |
| `ButtonInactive` `#555555` | Ближе к `#1A1A1A` + border. |
| `SeparatorColor` `#1A1A1A` | Для линии под кнопками — `#1F1F1F`; шапка может остаться отдельным токеном. |
| `TextMuted` `#AAAAAA` | Приглушить до `#666666` для соответствия макету. |
| Hover активной `#00E6C3` | Заменить на оранжевый hover. |

Ошибка/красная кнопка (`ButtonError`) в референсе не показана — оставить красный или слегка подстроить под новую гамму (отдельное решение при реализации).

## Скругления и геометрия

| Элемент | Референс | Текущий код (`PanelStyles`) |
|---------|----------|-----------------------------|
| Внешний контейнер панели | ~8–12 px (визуально «мягче» текущего) | `CornerRadius = 4` |
| Кнопки мини-панели | ~4 px | Задаётся у `Border` кнопок в API / стиль |
| Круг-маркер в шапке | круг `d ≈ 6–8 px` | Реализовать `Border` с `CornerRadius = половина стороны` или эквивалент |
| Круг статуса | малый, как маркер | Аналогично |

Рекомендация для реализации: вынести в `PanelStyles` константы `CornerRadiusPanel`, `CornerRadiusButton`, `CornerRadiusLarge`, чтобы основная и мини-панель были согласованы.

## Типографика

| Элемент | Референс | Примечание для cTrader Automate |
|---------|----------|----------------------------------|
| Семейство | Inter / Roboto / Segoe UI — чистый sans | В проекте **не используется** `FontFamily`; типично остаётся **системный шрифт платформы** (на Windows часто Segoe UI). Если в вашей версии API есть `FontFamily` у контролов — можно задать явно; иначе ограничиться `FontSize` и `FontWeight`. |
| Заголовок шапки | мелкий, ALL CAPS, muted | Сейчас `_titleText`: bold + `FontSizeNormal` — для макета: уменьшить размер (ориентир **10–12**), `FontWeight` обычный или SemiBold, цвет `TextMuted`, текст в верхнем регистре (или через локализацию). |
| Кнопки LM/MK/OK/FST | bold, белый, капс в макете | Уже `MiniButtonFontSize = 10`, bold — проверить единообразие с полной панелью после смены палитры. |
| Футер | обычный вес, muted, меньше кнопок | Новые `TextBlock`, размер ~10–11. |

## Отступы и сетка (ориентиры)

- Внутренние поля панели: **щедрые**, единые слева/справа (референс ~12–16 px по горизонтали; уточнить под `PanelWidth` 208).
- Зазор между кнопками мини-ряда: **~6–8 px** (сейчас в основном margin кнопок `2` — возможно увеличить для «воздуха» как на макете).
- Футер: `HorizontalAlignment` stretch, слева/справа подписи (аналог `space-between`).

---

## Поведение и контент (не только косметика)

- **Шапка мини-состояния:** текст вида «COP V1 — MINI PANEL» на референсе; в продукте — либо отдельная строка только в свёрнутом виде, либо обобщённый заголовок через `Localization` (RU/EN).
- **Футер:** «Свернутая панель» + «Активна» с зелёным индикатором — отражает состояние робота (запущен / готов). Нужны ключи локализации и правило: когда статус «не активен» (если бывает) — другой цвет/текст.
- **Полная панель (развёрнутая):** тот же токен цветов (`PanelStyles`), те же скругления и приглушённый вторичный текст, чтобы не было «два разных продукта».

## Ограничения API (на память)

- Нет «CSS flexbox» — только `StackPanel`, выравнивание, фиксированные ширины.
- Тени в макете минимальны; объём за счёт слоёв цвета и бордера — это совместимо с `Border` + `BackgroundColor` / `BorderColor`.
- Любые новые элементы (круги) — через маленькие квадратные `Border` с большим `CornerRadius` или доступные примитивы той версии API, которую вы используете.
