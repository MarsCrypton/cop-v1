# Стабильность линий SL/TP при смене таймфрейма (MAR-49)

## Связь

- **Linear:** [MAR-49](https://linear.app/marskorshun/issue/MAR-49/plagin-cop-pri-smene-tajmfrejma-propadaet-liniya-tejkstop-i-zavisaet)
- **Ветка Git:** `mar-49-timeframe-chart-lines`
- **Проект Linear:** Плагин COP (лимитки)

## Проблема

После включения режима с линиями на графике (в т.ч. через **MK** на мини-панели) при **смене таймфрейма** пропадают линии **stop / take** (и связанные подписи), плагин может **перестать отзываться** («зависание»).

## Ожидаемое поведение

- Линии и подписи **остаются корректными** после переключения ТФ (или **восстанавливаются** из сохранённых уровней).
- Панель и расчёты **не блокируют** терминал.

## Направление решения (черновик)

1. **Привязка подписей (`ChartText`)**  
   Сейчас текст часто ставится по **индексу бара** (`LastVisibleBarIndex + n`). При смене ТФ индексы и видимая область меняются — объекты могут оказаться вне экрана или в невалидном состоянии. В cTrader для оси X предпочтительно использовать **время** там, где API это позволяет ([Chart objects: bar index or time](https://help.ctrader.com/ctrader-algo/guides/ui-operations/chart-objects?q=)).

2. **Обновление после смены ТФ**  
   При перестройке графика — **перепозиционировать** подписи и при необходимости **пересоздать** линии из кэшированных цен (`ChartLineManager`), не полагаясь на «старые» индексы баров.

3. **Защита от каскадных обновлений**  
   Цепочка `Chart.ObjectsUpdated` → пересчёт → `RemoveObject` / `DrawText` может давать лишние срабатывания. Имеет смысл **флаг/debounce** или обновление текста без лишнего удаления, где возможно.

## Файлы (план правок — уточняются)

| Файл | Роль |
|------|------|
| `cop v1/Chart/ChartLineManager.cs` | Рисование линий и текста, подписка на `ObjectsUpdated` |
| `cop v1/COP.cs` | `RecalculateAll`, `OnTick`, жизненный цикл |
| `cop v1/Chart/FastOrderHandler.cs` | Тексты и линии в режиме Fast Order |

## Критерии готовности

- [ ] Сценарий: **MK** (или эквивалент) → линии на графике → переключение ТФ (например 5m ↔ 1m) **не ломает** отображение SL/TP.
- [ ] Панель остаётся отзывчивой, нет ощущения «зависания».
- [ ] При необходимости — отдельная проверка с включённым **Fast Order**.
