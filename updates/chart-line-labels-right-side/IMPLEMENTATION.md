# Подписи торговых линий: информационные блоки справа

## Связь

- **Linear:** [MAR-76](https://linear.app/marskorshun/issue/MAR-76/podpisi-u-linij-entrysltp-bloki-sprava-loty-market-sl-rr)
- **Ветка Git (предложение):** `mar-76-chart-line-labels-right-side`
- **Проект Linear:** Плагин COP (лимитки)

## Проверка формулировки (мы поняли одинаково)

Речь про сценарий, когда ты включаешь **режим с линиями на графике** через быстрые действия (**L** — Limit, **M** / **MK** — Market и т.п., как в панели): на экране **Entry / SL / TP** и **подписи** к ним (лоты, стоп в %, RR).

**Запрос автора:** сейчас эти подписи воспринимаются **слева** (относительно удобной зоны чтения / шкалы цен); нужно перенести **информационные блоки вправо** — к **правому краю** области графика (логически «у цен», в колонку справа), чтобы не мешали свечам слева и читались рядом со шкалой.

Это **то же самое**, что заложено в MAR-76 и в таблице ниже. Если ты имел в виду другое (например, якорь именно у **левого** последнего бара в окне — `FirstVisibleBar` — и текст вправо от него), это **другая геометрия**; тогда нужно явно уточнить в комментарии к задаче.

### Как в коде сейчас (важно для «слева / справа»)

- Якорь X: **`GetLabelAnchorTime`** → время бара с индексом **`Chart.LastVisibleBarIndex`**, то есть по оси времени это обычно **самый правый** видимый бар (свежая котировка в стандартной ориентации графика), а не «левый».
- Выравнивание: **`HorizontalAlignment.Left`** → **левый край** текста стоит в точке якоря, строка **продолжается вправо** от этого бара (часто в сторону **цены / правой кромки**). Из‑за этого визуально блок может казаться «в середине» или «не там», и не выглядит как **аккуратная колонка у правого края**.

Итог: мы трактуем твой «хочу **справа**» как **визуально прижать подписи к правой части графика** (корректировка `HorizontalAlignment` и при необходимости смещение якоря после теста в терминале), без смены режимов **L / M / MK**.

## Проблема

Подписи у линий **Entry / Stop Loss / Take Profit** создаются в **`ChartLineManager.DrawLineText`**: якорь по времени — **`GetLabelAnchorTime`** (время **последнего видимого бара**), выравнивание **`HorizontalAlignment.Left`**. См. раздел выше: при такой комбинации подписи **не выглядят** как **единая колонка информации справа** у края графика.

Требование: **справа** на графике отображать:


| Линия     | Содержание блока (смысл)                                                                               |
| --------- | ------------------------------------------------------------------------------------------------------ |
| **Entry** | Лоты **Market** или **Limit** (как сейчас по логике: `_lastVolumeLots` в тексте).                      |
| **SL**    | Информация о **стопе** (сейчас в `RecalculateAll` в подпись идёт только **процент** через `StopText`). |
| **TP**    | **RR** (сейчас в основную линию TP через `TpText` с `rr`).                                             |


## Текущий код (отправная точка)

- **`cop v1/Chart/ChartLineManager.cs`**
  - `DrawLineText` — `DrawText` + `HorizontalAlignment.Left`, `VerticalAlignment.Center`.
  - `GetLabelAnchorTime(Robot)` — время для X-якоря (устойчивость к ТФ, см. `updates/timeframe-chart-lines-stability`).
- **`cop v1/COP.cs`** — `RecalculateAll`, шаг «=== 9. Обновить тексты на линиях ===»:
  - Entry: `Localization.Get("LimitText" / "MarketText", лоты)`.
  - SL: `Localization.Get("StopText", slPercent)`.
  - Основной TP: `Localization.Get("TpText", rr)`.
- **`cop v1/UI/Localization.cs`** — ключи `LimitText`, `MarketText`, `StopText`, `TpText`.

Уже считаются в `RecalculateAll`, но **в подпись SL не передаётся**: `slDollars` (сумма риска в валюте счёта) — при желании добавить в строку SL на графике.

## Предлагаемое исправление

### 1. Выравнивание по правому краю видимой области

- Для подписей линий (`EntryTextId`, `SlTextId`, `Tp1/2/3TextId`) выставить **`HorizontalAlignment.Right`**.
- Якорь по времени **оставить** `GetLabelAnchorTime` (правый край видимых баров по времени): при `Right` **правый** край текста совпадает с якорем, тело строки уходит **влево** в область графика — колонка подписей оказывается **у правого края свечей**, ближе к шкале цен.

При необходимости (если визуально тесно): добавить **смещение якоря на 1 бар влево** по индексу (`LastVisibleBarIndex - 1` с проверкой границ) — только если после проверки в терминале так читабельнее.

### 2. Содержимое строк (минимальный объём)

- **Entry** — без изменения смысла; при необходимости уточнить формат строк в `Localization` (например, префикс «лоты» явнее).
- **SL** — расширить отображение: кроме `{0}%` добавить **сумму риска** (`slDollars`) в той же строке или второй строкой (если API `ChartText` допускает многострочный текст — проверить в cTrader; иначе одна строка с разделителем, напр. `STOP 1.2% · $50`).
- **TP** — оставить акцент на **RR**; при **2–3 тейках** решить явно:
  - либо RR только на **основной** TP-линии (как сейчас `MainTpTextId`),
  - либо краткий RR/доля на каждой линии — зафиксировать в задаче после UX-решения.

### 3. Точки вызова

- Все пересоздания подписей должны проходить через **`DrawLineText` / `UpdateLineTextPosition`**, чтобы не дублировать выравнивание.
- После правки выравнивания прогнать сценарии из **`timeframe-chart-lines-stability`**: смена ТФ, `RestoreLinesFromPrices`, `RecalculateAll`.

### 4. Локализация

- Новые шаблоны для SL (если добавляется `$` / валюта счёта) — новые ключи или расширение `StopText` с двумя плейсхолдерами `{0}` `%`, `{1}` деньги; переводы для всех языков в `Localization.cs`.

## Критерии приёмки (черновик)

- Подписи у линий визуально **прижаты к правой части** видимого графика и читаются без обрезки.
- На линии входа видны **лоты** (Market/Limit).
- На линии SL — **информация о стопе** (как минимум процент; опционально сумма риска).
- На линии TP — **RR** (согласовано для 1/2/3 тейков).
- Нет регрессии по MAR-49 (линии и подписи после смены ТФ / восстановления).

## Реализовано (2026-03-25, MAR-76)

- **`ChartLineManager.DrawLineText`** и **`FastOrderHandler.DrawFastText`**: `HorizontalAlignment.Right`, якорь по-прежнему `GetLabelAnchorTime`.
- **`StopText`**: два плейсхолдера `{0}` = процент риска, `{1}` = сумма в валюте счёта (`ChartLineManager.FormatRiskMoneyAccount`). Обновлены все вызовы в **`COP.RecalculateAll`**, **`HandleSlFieldChanged`**, **`ChartLineManager`** (первичная отрисовка / restore), **`FastOrderHandler`**.
- Сборка `dotnet build` — без ошибок. Ручную проверку в cTrader (ТФ, fast order, 1–3 TP) выполняет автор.
