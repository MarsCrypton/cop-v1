# Защита кода cTrader / cAlgo: что реально даёт платформа

Документ для автора алгоритма (cBot / индикатор), который хочет **показывать продукт другим**, но **не отдавать исходный C#**.

Связанная задача в Linear: **MAR-75** (проект «Плагин COP (лимитки)»).

---

## 1. Важно понимать с самого начала

- **«Скомпилировать так, чтобы никто не смог увидеть код»** в смысле *математической невозможности восстановить логику* — так не бывает ни в cTrader, ни в .NET, ни где либо: достаточно мотивированный специалист может пытаться анализировать поведение и защищённые форматы.
- **Реалистичная цель:** чтобы обычный пользователь и случайный «скачал исходники» **не получили ваш `.cs`**, а дистрибуция шла через **официальный механизм платформы** (сборка → `.algo` → экспорт без кода / Store).

Имеет смысл сочетать **технические меры cTrader** с **юридическими** (лицензионное соглашение, запрет на декомпиляцию/распространение — по консультации с юристом) и **организационными** (кому вы доверяете исходники).

---

## 2. Что такое «компиляция» в cTrader

В контексте **cTrader Algo** (роботы и индикаторы на C# или Python):

1. Вы пишете код **локально** в cTrader Automate (Windows или Mac).
2. При **сборке (Build)** платформа создаёт файл `**.algo`** — это **не обычная .NET DLL** с CIL-байткодом для типичных декомпиляторов.
3. Официально заявляется, что `.algo`:
  - выполняется **в памяти**, а не как «читаемый» исполняемый файл с диска;
  - **не содержит стандартного CIL**, на котором заточены многие .NET-декомпиляторы;
  - поддерживает **опциональную обфускацию** при сборке;
  - спроектирован с **устойчивостью к типичной декомпиляции**.

Источник: [Algo security measures](https://help.ctrader.com/ctrader-algo/documentation/protection-measures/) (официальный справочник cTrader).

**Итог для друзей:** им нужно передавать **готовый `.algo` (без папки с `.cs`)**, а не репозиторий или архив с исходниками.

---

## 3. Экспорт: главный рычаг, когда вы не продаёте через Store

После сборки алгоритм можно **экспортировать**:

- **С исходным кодом** — только для доверенных людей (совместная разработка).
- **Без исходного кода** — **рекомендуемый вариант** для широкой или полудоверенной аудитории.

При экспорте **без кода** можно настроить права **только на запуск (run-only)**:


| Режим                               | Смысл                                                                                                           |
| ----------------------------------- | --------------------------------------------------------------------------------------------------------------- |
| **Доступ конкретному пользователю** | Запуск только у указанного **cTrader ID** — удобно для «показать другу»: вы привязываете сборку к его аккаунту. |
| **Доступ любому пользователю**      | Любой, у кого есть файл, может **запускать**, но **исходников в экспорте нет**.                                 |


Подробности: [Export and install algos](https://help.ctrader.com/ctrader-algo/how-tos/all-algos/export-and-install-algos/).

**Практика «показать друзьям, но не дать код»:**

1. Собрать робота в cTrader (**Build**).
2. **Экспортировать без исходного кода**.
3. По возможности выбрать **привязку к cTrader ID друга** (если цель — один человек, а не публичная раздача).

## 4. cTrader Store (если пойдёте в продажу или публичный каталог)

При публикации в **cTrader Store**:

- загружаемый `.algo` **шифруется**;
- даже если вы загрузили экспорт «с исходниками», в опубликованном продукте **исходники удаляются** — остаётся защищённая версия;
- покупателю выдаётся **уникальная копия**, привязанная к **cTrader ID**, файлы **нельзя передавать между пользователями** как равнозаменяемую копию (в рамках модели лицензирования Store).

Источники: [Publish a product](https://help.ctrader.com/ctrader-store/how-tos/publish-a-product/), тот же раздел [protection measures](https://help.ctrader.com/ctrader-algo/documentation/protection-measures/).

---

## 5. Обфускация

- В экосистеме cTrader для `.algo` описывается **встроенная опциональная обфускация при сборке** — имеет смысл **включить её** в настройках сборки, если платформа предлагает (уточняйте в актуальной версии клиента и справке).
- Сторонние .NET-обфускаторы применимы **осторожно**: цепочка сборки у cAlgo специфична; неправильная обфускация может сломать робота. Любые сторонние инструменты — только после проверки на **копии проекта** и полном прогоне в демо.

---

## 6. Чего не делать, если боитесь копирования

- Не отправлять **ZIP с `.cs`**, `.csproj`, `.sln** — это и есть ваши исходники.
- Не выкладывать в публичные репозитории **без лицензии и осознанного решения**.
- Не полагаться на «секретность через безопасность через неясность» одной лишь логики — **дистрибуция через `.algo` без исходников** надёжнее для типичного сценария.
