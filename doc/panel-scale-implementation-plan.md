# COP v1 — план обновления: масштаб интерфейса панели

## Цель

Добавить настройку масштаба панели COP v1, чтобы пользователь мог увеличивать или уменьшать весь интерфейс плагина.

Ожидаемое поведение:

- новая настройка в параметрах cBot (окно настроек, где сейчас `Vertical Position`, `Horizontal Position`, `Panel transparency %`);
- масштаб применяется ко всей панели (развёрнутой и мини-версии);
- масштаб сохраняется между перезапусками;
- визуал остаётся пропорциональным (шрифты, кнопки, отступы, ширина, высоты).

---

## Анализ текущей архитектуры (по проекту)

### Где сейчас задаются размеры

1. `cop v1/UI/PanelStyles.cs`
  - хранит базовые константы размеров (`PanelWidth`, `FontSize*`, `HeaderBarButtonHeight`, отступы для мини-кнопок);
  - содержит методы применения стилей для кнопок и полей.
2. `cop v1/UI/MainPanel.cs`
  - строит панель вручную через `StackPanel`, `Grid`, `Border`;
  - использует много фиксированных значений `Width`, `Height`, `Margin`, локальные `const`;
  - часть размеров привязана к `PanelStyles.PanelWidth`, часть задана напрямую числами.
3. `cop v1/COP.cs`
  - объявляет параметры плагина через `[Parameter(...)]`;
  - создаёт `MainPanel` и передаёт в конструктор `panelTransparencyPercent`;
  - уже реализует чтение/сохранение части UI-настроек через `LocalStorage`.

### Вывод по реализации

Изменение масштаба нужно делать через отдельный "коэффициент UI" и централизованную функцию масштабирования, иначе правка множества отдельных чисел приведёт к регрессиям.

---

## Ограничения и рамки

1. UI на `cAlgo.API` (не HTML/CSS), нет "настоящего responsive layout".
2. Много фиксированных пиксельных значений — без рефакторинга в helper-функции масштаб будет неполным.
3. Важно сохранить совместимость текущей логики:
  - торговые события не должны измениться;
  - локализация и поведение кнопок должны остаться прежними.
4. Параметры cBot должны быть на английском (принято в проекте).

---

## Предлагаемая модель масштаба

### Формат настройки

- Параметр: `Panel scale %`
- Тип: `int`
- Диапазон: `70..150` (рекомендуемый безопасный коридор)
- Шаг: `10`
- Значение по умолчанию: `100`

### Правило масштабирования

- `scale = panelScalePercent / 100.0`
- Все размеры панели получают значение через helper:
  - `S(px)` -> масштабированный размер;
  - `SF(fontSize)` -> масштабированный размер шрифта (минимум 8-9 для читаемости);
  - `ST(thickness)` -> масштабированные отступы.

### Что входит в "весь плагин"

- ширина панели;
- высоты и ширины кнопок;
- размеры шрифтов;
- отступы/margin/padding;
- размеры мини-панели и футера.

Не входит:

- торговые уровни и расчёты риска/объёма;
- бизнес-логика ордеров.

## Пошаговый план реализации

Ниже шаги сделаны так, чтобы каждый можно было выполнить отдельно другим ИИ.

### Шаг 1. Подготовить инфраструктуру масштабирования в `PanelStyles`

**Задача:**
добавить API масштабирования, не ломая текущий внешний вид при масштабе 100%.

**Что сделать:**

1. Добавить текущий масштаб (например, `double UiScale = 1.0`) и сеттер `SetScalePercent(int percent)`.
2. Добавить helper-методы:
  - `double S(double value)`
  - `int SI(int value)` (округление)
  - `Thickness ST(double left, double top, double right, double bottom)` и overload для одного значения
  - `int SF(int fontSize)` (масштаб шрифтов с минимумом).
3. Перевести константы размеров в свойства, рассчитываемые от базовых значений.
4. Сохранить обратную совместимость: при 100% размеры совпадают с текущими.

**Промт для другого ИИ:**

```text
Ты работаешь в проекте COP v1 (cTrader, C#). Реализуй в файле cop v1/UI/PanelStyles.cs инфраструктуру масштабирования UI.

Требования:
1) Добавь поддержку panel scale в процентах (70..150), внутренний коэффициент scale = percent / 100.0.
2) Добавь helper-методы:
   - S(double) -> double
   - SI(int) -> int
   - SF(int) -> int (с ограничением минимального шрифта не ниже 8)
   - ST(...) -> Thickness
3) Переведи ключевые размеры PanelStyles (ширина панели, шрифты, высоты кнопок, скругления и т.д.) на расчёт через scale.
4) При scale=100 внешний вид должен остаться максимально идентичным текущему.
5) Не меняй торговую логику и не трогай другие файлы.

После правки проверь компиляцию решения и дай краткий отчёт о том, какие поля/свойства стали масштабируемыми.
```

### Шаг 2. Добавить параметр `Panel scale %` в `COP.cs` и хранение в `LocalStorage`

**Задача:**
дать пользователю настройку масштаба через стандартное окно параметров cBot.

**Что сделать:**

1. Добавить параметр:
  - `[Parameter("Panel scale %", Group = "Panel alignment", DefaultValue = 100, MinValue = 70, MaxValue = 150)]`
2. Добавить `LoadSavedScalePercent()` / `SaveScalePercent(int percent)` по аналогии с прозрачностью.
3. Определить итоговый scale при старте:
  - если в storage нет значения -> брать параметр;
  - если параметр отличается от сохранённого -> считать параметр новым source of truth и перезаписать storage;
  - иначе брать сохранённое.
4. Перед созданием `MainPanel` вызвать `PanelStyles.SetScalePercent(scalePercent)`.

**Промт для другого ИИ:**

```text
В проекте COP v1 добавь в COP.cs поддержку настройки "Panel scale %" в группе "Panel alignment".

Требования:
1) Новый параметр:
   [Parameter("Panel scale %", Group = "Panel alignment", DefaultValue = 100, MinValue = 70, MaxValue = 150)]
2) Реализуй LoadSavedScalePercent/SaveScalePercent через LocalStorage (scope Device), по аналогии с прозрачностью.
3) На OnStart вычисли итоговый scale с логикой приоритета:
   - нет сохранённого -> параметр;
   - сохранённое есть, но параметр изменён пользователем -> берём параметр и сохраняем;
   - иначе берём сохранённое.
4) До создания MainPanel вызови PanelStyles.SetScalePercent(finalScalePercent).
5) Не меняй торговые расчёты и обработчики ордеров.

После правки покажи кратко: где добавлен параметр, какие методы LocalStorage добавлены, где применяется scale при старте.
```

**Задача:**

**Что сделать:**

1. В `MainPanel` добавить `ComboBox` "Panel scale" (70..150, шаг 10).
2. Добавить событие `OnScaleChanged(int percent)`.
3. В `COP.cs` подписаться на событие:

## Риски и как их снизить

1. **Обрезка текста на больших масштабах**
  - закладывать запас по ширине кнопок;
  - при необходимости уменьшать font-size отдельных коротких кнопок (например, header button).
2. **Съезжающая геометрия мини-панели**
  - вычислять ширины только от масштабированного `PanelWidth`;
  - минимизировать "магические числа".
3. **Мерцание при live-изменении масштаба**
  - использовать controlled recreate панели с восстановлением состояния.
4. **Регрессия поведения**
  - не смешивать рефакторинг торговой логики с UI-масштабированием.
